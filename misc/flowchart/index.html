BeginBattle

  1. Left-click and drag a box to select all combatants
  2. Right-click on one combatant and toggle combat state
  3. Switch to the combat tab
  4. Click "Roll All" to determine combat order. Those with highest basic speed go first [BX363]. In the case of a tie, the character with the highest DX can go first [BX363]. If there is still a tie, GM can choose (GURPS game aid uses 1d6).
  5. Click "Begin Combat" (at the bottom)

Attack

  1. Check if attack is successful
    1. If weapon is not ready
      1. If Fast-Draw Skill [B194] exists, click on RSL button in Character Sheet and check chat window for result. On success, proceed with attack.
      2. On failure or if no Fast-Draw Skill exists, spend this turn executing Ready maneuver and attack next turn.
    2. If ranged attack
      1. If aiming on previous turns
        1. If choosing new target, discard aiming modifier
        2. Otherwise, click on bucket and add recorded aiming modifier
      2. Add range modifier to bucket: Ctrl-click-drag a line to the target, then click on target.
    3. Perform the attack. Click on Level button in Weapon table in Character Sheet and check chat window for result.
    4. If attack is successful, move to next step. Otherwise, turn ends.
  2. Check if defense is successful
    1. No active defense is possible if the defender is unaware of the attack, unconscious, immobilized, or otherwise unable to react [BX374]. In this case, assume defense fails and move to next step (calculate damage).
    2. Select the combatant and open the modifier bucket dialog. Click on all buttons in the "Current Effects" section that are related to active defenses.
    3. If defender's last maneuver was All-Out Defense (increased defense for block, parry, or dodge) [BX366], and defending with the method chosen at the time of the maneuver, then apply the bonus to the bucket (may do this when defending against every attack this turn).
    4. If blocking
      1. Any ready object (shield, cloak, or other large object, including another character) may be used to block [BX375]. If object is not ready, defender cannot use this defense and must spend a turn executing Ready maneuver to use this defense on future turns.
      2. You can block any melee attack, thrown weapon, projected liquid, or muscle-powered missile weapon. You cannot block bullets or beam weapons (too fast) [BX375].
      3. click on Block button in Melee Weapon table in Character sheet and check chat window for result
    5. If parrying
      1. The defender can parry melee attacks or thrown weapons. Other ranged weapons can be parried only if the attacker is within reach of the defender's melee weapon [BX376].
        • Thrown weapons larger than 1 lb. (e.g. spears) are parried with a -1 penalty [BX376] (add this to the bucket)
        • Thrown weapons smaller than 1 lb. (e.g. knives, shuriken) are parried with a -2 penalty [BX376] (add this to the bucket)
      2. Each successive parry this turn carries a cumulative penalty (add this to the bucket):
        • -4 Normally
        • -2 with either the Trained by a Master [B93] or Weapon Master [B99] advantage
        • -1 with both of these advantages
      3. click on Parry button in Melee Weapon table in Character sheet and check chat window for result
      4. If an unarmed attack was successfully parried, the defender gets a free attack on the attacker. Immediately roll for attack (-4 penalty if attacker used Judo or Karate [B203]) and damage. There is no defense. [BX376]
    6. If dodging
      1. Dodge may be used to defend against any attack [BX374]
      2. click on Dodge button in Encumberance, Move, and Dodge table in Character sheet and check chat window for result
    7. If the defender's last maneuver was All-Out Defense (double defense) [BX366], and if the previous defense was unsuccessful, the defender may try to defend a second time with a different ready active defense (may do this when defending against every attack this turn).
  3. Calculate damage and update defender status
    1. Right-click on defender and toggle target status on.
    2. Click on the box to the right of "Roll damage" in the chat message that showed the attack was successful
    3. Click "Apply All to..." button in the chat message showing the damage roll. The targeted unit should be the unit listed.
      • All modifiers (damage reduction, wound modifiers, and armor divisors) should be calculated automatically
      • If penetrating damage is greater than zero, then "Shock" will appear under "Calculation and Results". Click on the chat box icon next to "Shock", then click on button in the chat message (must do both). Should see the defender's shock status modifier , , , or set, depending on the amount shown.
      • If penetrating damage is greater than half of total HP, then "Major Wound" will appear under "Calculation and Results". Click on the chat box icon next to "Major Wound", then click on button in the chat message to roll to avoid Stun/Knockdown. If unsuccessful, then the Stunned and Lying Prone statuses will be set.
      • Click "Apply (Quietly)" in the Apply Damage Dialog. Should see the HP (or FP) value decreased in the defender's status.
      • Toggle the Reeling status indicator if the combatant's HP has been reduced to 1/3 of its full value. You should see the Reeling label in the Hit Points table of the combatant's character sheet, but no modifiers will be applied (and it will not pulse) unless you toggle the status indicator.
    4. Right-click on defender and toggle target status off (otherwise next attack will be applied to the same defender).

Stunned

  • If a combatant receives a major wound and fails an HT roll, it is stunned [BX381,420].
  • Effects
    • -4 to all active defenses
    • Only allowed maneuver is Do Nothing
  • To remove stun condition, make a successful HT roll at the end of combatant's turn.
  • The stun condition continues for another turn if the HT roll is unsuccessful.
  • GURPS Combat assistant does not add this effect automatically, but it does suggest it in the "Current Effects" section of the modifier bucket dialog.

Shocked

  • If a combatant takes any damage, it is shocked for one turn.
  • Shock is a penalty to DX, IQ, and skills based on those attributes on your next turn (only) [BX381].
  • Apply the penalty (between -1 and -4) to the bucket before performing any DX or IQ roll. The penalty is not applied automatically, but should see a button in the "Current Effects" section of the Modifier Bucket Dialog if the status is set on the token.
  • Make sure that the condition expires at the end of the combatant's turn (hopefully this will happen automatically).

Reeling

  • Once a combatant has been reduced to 1/3 of their original HP, the combatant is reeling [BX380,419].
  • Effects
    • Basic speed is halved (round up)
    • Move is halved (round up)
    • Dodge is halved (round up)
  • The GURPS Combat Assistant will automatically show whether or not a combatant is reeling in the Hit Points table of the character sheet, but it will not automatically apply the effects.
  • To apply the effect, toggle the Reeling status indicator on the token. You will notice that the speed, move, and dodge are reduced. You will also notice that the reeling indicator in the Hit Points table starts to pulse.
